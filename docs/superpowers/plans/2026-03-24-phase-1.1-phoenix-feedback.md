# Phase 1.1: Phoenix Feedback Improvements

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Address three concrete friction points discovered during Phoenix's adoption of Orthrus.

**Architecture:** Three independent improvements to Orthrus's public API: (1) re-export accesskit so consumers don't need a direct transitive dependency, (2) add a `try_snapshot()` wrapper on `TestHarness` that detects headless/no-GPU environments and returns `Err` instead of panicking, (3) expand `kittest_prelude` to re-export `accesskit::Role` and document the `get_by_role()` pattern.

**Tech Stack:** Rust, egui 0.31, egui_kittest 0.31, accesskit 0.17

**Beads:** orthrus-6w7 (accesskit re-export), orthrus-2if (snapshot degradation), orthrus-860 (get_by_role docs)

### Task 2: Graceful snapshot degradation in headless environments (orthrus-2if)

**Files:**
- Modify: `src/harness.rs`
- Modify: `src/lib.rs`
- Modify: `tests/counter_tests.rs`

**Context for implementor:** The "No adapter found" panic comes from `egui_kittest`'s `WgpuTestRenderer::new()` which calls `create_render_state(default_wgpu_setup())`. The `default_wgpu_setup()` native_adapter_selector returns `Err("No adapter found")` when no GPU adapters exist, and `create_render_state` calls `.expect()` on it. This happens at renderer creation time, not at render time. `try_snapshot()` on `egui_kittest::Harness` already returns `Result<(), SnapshotError>`, but it panics before reaching that point because the renderer itself can't be created.

The fix: `TestHarness` should provide its own `try_snapshot()` that catches the renderer creation failure and returns it as an `Err` rather than letting it panic. We can detect GPU availability by attempting to enumerate wgpu adapters.

**Important:** `egui_kittest::Harness::try_snapshot()` is only available when the `wgpu` feature is enabled. Our `TestHarness::try_snapshot()` should also be gated on the `kittest` feature (which already enables wgpu on egui_kittest).

- [ ] **Step 1: Write failing test — `try_snapshot` returns Err in headless**

Add to `tests/counter_tests.rs`:

```rust
#[test]
fn try_snapshot_returns_result() {
    let mut harness = TestHarness::<CounterApp>::new();
    harness.run();

    // try_snapshot should return a Result, not panic, regardless of GPU availability.
    // In headless environments this will be Err; with GPU it will be Ok or Err(diff).
    // The key contract: it NEVER panics.
    let result = harness.try_snapshot("counter_try_snapshot_test");
    // We just verify it returns a Result (doesn't panic)
    let _ = result;
}
```

- [ ] **Step 2: Run test to verify behavior**

Run: `cargo test --features kittest try_snapshot_returns_result 2>&1`

**Environment-dependent behavior:** In headless (no GPU), this test will PANIC via Deref to `egui_kittest::Harness::try_snapshot()` — confirming the problem we're fixing. In a GPU environment, it will pass via Deref (the existing method works fine with a GPU). If you're in a GPU environment and the test passes, that's OK — proceed to Step 3 anyway. The fix is still needed for headless consumers.

- [ ] **Step 3: Add `try_snapshot()` to `TestHarness` that handles missing GPU**

In `src/harness.rs`, add the snapshot method. We need to re-export `SnapshotError` from egui_kittest first.

In `src/lib.rs`, add to the kittest-gated section:
```rust
#[cfg(feature = "kittest")]
pub use egui_kittest::SnapshotError;
```

In `src/harness.rs`, add this method to the `impl<A: TestableApp + 'static> TestHarness<A>` block:

```rust
    /// Takes a snapshot, returning `Err` instead of panicking when no GPU is available.
    ///
    /// In headless or no-GPU environments (CI, dev containers), the underlying wgpu
    /// renderer cannot initialize. This method catches that failure and returns it as
    /// a `SnapshotError::RenderError` rather than panicking.
    pub fn try_snapshot(&mut self, name: &str) -> Result<(), egui_kittest::SnapshotError> {
        // Attempt to catch panics from renderer initialization (no GPU adapter)
        // by using the inner harness's try_snapshot which returns Result.
        // However, egui_kittest's try_snapshot still panics if the renderer
        // can't be created. We use std::panic::catch_unwind to convert that to Err.
        let result = std::panic::catch_unwind(std::panic::AssertUnwindSafe(|| {
            self.inner.try_snapshot(name)
        }));

        match result {
            Ok(inner_result) => inner_result,
            Err(panic_payload) => {
                let msg = if let Some(s) = panic_payload.downcast_ref::<String>() {
                    s.clone()
                } else if let Some(s) = panic_payload.downcast_ref::<&str>() {
                    (*s).to_string()
                } else {
                    "snapshot rendering panicked (likely no GPU adapter available)".to_string()
                };
                Err(egui_kittest::SnapshotError::RenderError { err: msg })
            }
        }
    }
```

- [ ] **Step 4: Run test to verify it passes**

Run: `cargo test --features kittest try_snapshot_returns_result 2>&1`
Expected: PASS — the method now catches the panic and returns Err

- [ ] **Step 5: Update existing `snapshot_default_state` test for headless compatibility**

The existing test calls `harness.try_snapshot("counter_default").expect("snapshot comparison failed")`. Once our inherent `try_snapshot()` shadows the Deref version, it will return `Err(RenderError)` in headless instead of panicking — but `.expect()` will still panic with a different message. Update the test to handle headless gracefully:

```rust
#[test]
fn snapshot_default_state() {
    let mut harness = TestHarness::<CounterApp>::new();
    harness.run();

    // try_snapshot returns Err in headless environments (no GPU adapter).
    // In GPU environments, first run creates the reference image; subsequent runs compare.
    match harness.try_snapshot("counter_default") {
        Ok(()) => {} // Snapshot matched or was created
        Err(orthrus::SnapshotError::RenderError { .. }) => {} // No GPU — acceptable
        Err(e) => panic!("snapshot comparison failed: {e}"),
    }
}
```

Run: `cargo test --features kittest snapshot_default_state 2>&1`
Expected: PASS in both GPU and headless environments
